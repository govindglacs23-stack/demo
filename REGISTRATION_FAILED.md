# ❌ Registration Failed - Fix Guide\n\n## What This Means\n\nThe frontend is working ✅ but can't connect to the backend ❌\n\n---\n\n## 🚨 Quick Diagnosis\n\nOpen a new Command Prompt and run:\n\n```bash\ncurl http://localhost:5000/api/health\n```\n\n### Results:\n\n**If you see:**\n```json\n{\"success\":true,\"message\":\"Server is running\"}\n```\n→ Backend is running ✅ Skip to \"MongoDB Issues\" section\n\n**If you see:**\n```\nCould not resolve host localhost\nConnection refused\n```\n→ Backend is NOT running ❌ Follow \"Backend Not Running\" section\n\n---\n\n## ❌ Backend Not Running\n\n### Fix:\n\n1. **Open a new Command Prompt window**\n\n2. **Navigate to backend:**\n```bash\ncd C:\\Users\\gopal\\OneDrive\\Desktop\\LMS\\backend\n```\n\n3. **Start backend:**\n```bash\nnpm run dev\n```\n\n4. **You should see:**\n```\nServer running on port 5000\nMongoDB Connected: localhost\n```\n\n### If Backend Won't Start:\n\n#### Error: \"npm run dev not found\"\n→ Run: `npm install`\n\n#### Error: \"Cannot find module\"\n→ Run: `npm install`\n\n#### Error: \"EADDRINUSE: address already in use\"\n→ Port 5000 is taken. Try changing it in `.env`:\n```\nPORT=5001\n```\n\n#### Error: MongoDB connection error\n→ See \"MongoDB Issues\" section below\n\n---\n\n## 🗄️ MongoDB Issues\n\n### Check if MongoDB is Running:\n\nOpen new Command Prompt and run:\n```bash\nmongosh\n```\n\nIf it connects → MongoDB is running ✅\nIf it fails → MongoDB not running ❌\n\n### Fix MongoDB Connection:\n\n#### Option A: Start Local MongoDB\n\n**If you installed MongoDB locally:**\n\n1. Open new Command Prompt\n2. Run:\n```bash\nmongod\n```\n\n3. Leave it running\n4. You should see:\n```\nWaiting for connections on port 27017\n```\n\n#### Option B: Use MongoDB Atlas (Cloud)\n\nIf using cloud MongoDB:\n\n1. Go to https://www.mongodb.com/cloud/atlas\n2. Get your connection string\n3. Open `backend/.env`\n4. Change:\n```\nMONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/lms\n```\n5. Save and restart backend\n\n---\n\n## 📋 Complete Checklist\n\nBefore trying registration again, verify ALL of these:\n\n### ✅ Backend Requirements\n- [ ] Command Prompt window #1: Backend running (`npm run dev`)\n  - Should say \"Server running on port 5000\"\n  - Should say \"MongoDB Connected\"\n- [ ] `.env` file exists in backend folder\n- [ ] `.env` has correct MongoDB URI\n- [ ] `.env` has JWT_SECRET set\n\n### ✅ MongoDB Requirements\n- [ ] MongoDB is running (either local `mongod` or Atlas)\n- [ ] Connection string in `.env` is correct\n- [ ] If using Atlas: IP is whitelisted\n\n### ✅ Frontend Requirements\n- [ ] Command Prompt window #2: Frontend running (`npm run dev`)\n  - Should say \"VITE v... ready in ... ms\"\n- [ ] Browser shows http://localhost:3000\n- [ ] No errors in browser console (F12)\n\n---\n\n## 🔧 Step-by-Step Setup\n\n### Step 1: Setup MongoDB\n\n**Choose ONE option:**\n\n**Option A: Local MongoDB (Windows)**\n```bash\n# Download from: https://www.mongodb.com/try/download/community\n# Install it\n# Then in Command Prompt:\nmongod\n```\n\n**Option B: MongoDB Atlas (Cloud - Easier)**\n```\n1. Look for errors like:\n   - \"Cannot reach backend\"\n   - \"Network error\"\n   - \"CORS error\"\n\n### Check Network Tab (F12):\n\n1. Press **F12** in browser\n2. Go to **Network** tab\n3. Try registration\n4. Look for request to `http://localhost:5000/api/auth/register`\n5. Check if:\n   - Status is 500 (server error)\n   - Status is 0 (can't reach server)\n   - Response shows error message\n\n### Check Backend Terminal:\n\nLook for error messages when you click register:\n\n**Good sign:**\n```\nPOST /api/auth/register 201\n```\n\n**Bad sign:**\n```\nError: Cannot connect to MongoDB\nError: listen EADDRINUSE\nMongoNetworkError\n```\n\n---\n\n## 🐛 Specific Error Messages\n\n### \"Cannot reach server\" / \"Connection refused\"\n```\n❌ Backend not running\n✅ Run: npm run dev in backend folder\n```\n\n### \"MongoDB connection error\"\n```\n❌ MongoDB not running or wrong connection string\n✅ Run: mongod (for local)\n✅ Or verify Atlas connection string\n```\n\n### \"Email already in use\"\n```\n✅ This is GOOD! Means connection works\n✅ Use a different email address\n```\n\n### \"CORS error\"\n```\n❌ Backend CORS not configured correctly\n✅ Check backend/.env has:\n   FRONTEND_URL=http://localhost:3000\n```\n\n### \"Invalid request\" / \"Validation error\"\n```\n✅ Connection works but data format is wrong\n✅ Make sure you filled all fields\n✅ Password should be at least 6 characters\n```\n\n---\n\n## 💡 Pro Troubleshooting Tips\n\n### \"It was working before, now it's not\"\n\n1. Check if you closed the backend terminal\n2. Check if MongoDB stopped running\n3. Restart both:\n   ```bash\n   # Terminal 1\n   cd backend && npm run dev\n   \n   # Terminal 2\n   cd frontend && npm run dev\n   ```\n\n### \"Port already in use\"\n\n```bash\n# Find what's using port 5000\nnetstat -ano | findstr :5000\n\n# Kill it\ntaskkill /PID <PID> /F\n\n# Or change port in backend/.env\nPORT=5001\n```\n\n### \"Still getting error\"\n\n1. Restart your computer (clears ports)\n2. Delete node_modules and reinstall:\n   ```bash\n   rmdir /s node_modules\n   del package-lock.json\n   npm install\n   npm run dev\n   ```\n\n---\n\n## 📝 Final Checklist Before Registration\n\n```\nBefore attempting registration, verify:\n\n☐ Backend terminal shows:\n  \"Server running on port 5000\"\n  \"MongoDB Connected\"\n\n☐ Frontend terminal shows:\n  \"VITE ... ready in ... ms\"\n  \"Local: http://localhost:3000\"\n\n☐ Browser can reach:\n  http://localhost:3000 (shows app)\n  http://localhost:5000/api/health (returns JSON)\n\n☐ MongoDB is running:\n  mongod is running in separate terminal\n  OR Atlas connection is correct\n\n☐ .env file has:\n  MongoDB URI\n  JWT_SECRET\n  PORT=5000\n```\n\n---\n\n## 🎯 Most Common Fix\n\n**99% of registration failures are because:**\n\n1.
