# Frontend Not Showing - Troubleshooting Guide\n\n## ❌ Problem: Blank Page on localhost:3000\n\n### Quick Fixes (Try These First)\n\n#### 1. **Clear npm cache and reinstall**\n```bash\ncd frontend\nrmdir /s node_modules\ndel package-lock.json\nnpm cache clean --force\nnpm install\nnpm run dev\n```\n\n#### 2. **Check Console for Errors**\n- Press `F12` in browser (opens Developer Tools)\n- Go to **Console** tab\n- Look for red error messages\n- Screenshot them and check against solutions below\n\n#### 3. **Hard Refresh Browser**\n- Press `Ctrl + Shift + R` (Force refresh)\n- Or `Ctrl + F5`\n- Wait 5-10 seconds\n\n#### 4. **Check if Port 3000 is Running**\n- Go to http://localhost:3000\n- Check if browser shows connection refused\n- If yes, frontend isn't running\n\n### If Still Not Working\n\n## 🔍 Detailed Troubleshooting\n\n### Issue 1: \"Cannot GET /\"\n**Solution:** The frontend server isn't responding properly\n\n1. Kill the process: `Ctrl + C` in the terminal\n2. Delete node_modules: `rmdir /s node_modules`\n3. Delete package-lock.json: `del package-lock.json`\n4. Reinstall: `npm install`\n5. Run: `npm run dev`\n\n### Issue 2: \"Failed to resolve module\"\n**Solution:** Missing dependencies\n\nRun these commands:\n```bash\nnpm install react react-dom\nnpm install react-router-dom\nnpm install axios\nnpm install -D tailwindcss postcss autoprefixer\nnpm install react-icons\n```\n\n### Issue 3: Blank White/Black Page with No Errors\n**Solution:** Styling issue or React not rendering\n\n1. Check browser console (F12)\n2. Look for CSS errors\n3. Try clearing CSS cache:\n   ```bash\n   del frontend\\dist\\*\n   npm run dev\n   ```\n\n### Issue 4: \"Module not found\" errors\n**Solution:** Files are missing or have wrong paths\n\n1. Check all these files exist:\n   ```\n   frontend/src/App.jsx\n   frontend/src/main.jsx\n   frontend/src/index.css\n   frontend/index.html\n   ```\n\n2. If missing, they were created. Check the FILES_CREATED.md\n\n### Issue 5: Tailwind CSS Not Working (No Styling)\n**Solution:** PostCSS/Tailwind not configured\n\n1. Ensure these files exist:\n   - `frontend/tailwind.config.js`\n   - `frontend/postcss.config.js`\n   - `frontend/src/index.css` with `@tailwind` directives\n\n2. Reinstall Tailwind:\n   ```bash\n   npm install -D tailwindcss postcss autoprefixer\n   ```\n\n3. Restart dev server:\n   ```bash\n   Ctrl + C\n   npm run dev\n   ```\n\n### Issue 6: \"API Backend Connection Error\"\n**Solution:** Backend not running or wrong URL\n\n1. Check backend is running on http://localhost:5000/api/health\n2. If not, start backend in separate terminal:\n   ```bash\n   cd backend\n   npm run dev\n   ```\n\n3. If URL is wrong, edit `frontend/src/services/api.js`:\n   ```javascript\n   const API_BASE_URL = 'http://localhost:5000/api';\n   ```\n\n## 🛠️ Step-by-Step Complete Reset\n\n**If nothing else works, do a complete reset:**\n\n### Step 1: Stop all processes\n- Press `Ctrl + C` in all terminals\n- Close all browser windows\n\n### Step 2: Clean frontend\n```bash\ncd C:\\Users\\gopal\\OneDrive\\Desktop\\LMS\\frontend\nrmdir /s /q node_modules\ndel package-lock.json\ndel *.log\n```\n\n### Step 3: Reinstall everything\n```bash\nnpm cache clean --force\nnpm install\n```\n\n### Step 4: Start fresh\n```bash\nnpm run dev\n```\n\n### Step 5: Test in browser\n- Go to http://localhost:3000\n- Wait 10-15 seconds for Vite to compile\n- Press F5 to refresh\n\n## 📋 Checklist Before Running\n\n- [ ] Node.js installed (`node --version` works)\n- [ ] npm working (`npm --version` shows version)\n- [ ] Backend running on port 5000\n- [ ] MongoDB running\n- [ ] frontend folder has node_modules\n- [ ] Frontend package.json exists\n- [ ] All config files exist (vite.config.js, tailwind.config.js, postcss.config.js)\n\n## 🖥️ System Requirements\n\n- Node.js v14+ (https://nodejs.org/)\n- npm v6+\n- 512MB RAM minimum\n- Windows 7+ / macOS 10.12+ / Linux (any recent)\n\n## 📝 What to Look for in Browser Console\n\n**Good signs:**\n```\n✓ No red errors\n✓ Shows \"Vite dev server is running\"\n✓ React elements render\n```\n\n**Bad signs:**\n```\n✗ \"Cannot find module\"\n✗ \"CORS error\"\n✗ \"Cannot reach backend\"\n✗ Network tab shows failed requests\n```\n\n## 🎯 If Page Loads But Has No Styling\n\n1. **CSS not loading:**\n   - Check Network tab (F12)\n   - Look for 404 errors on CSS files\n   - Clear browser cache (Ctrl + Shift + Delete)\n\n2. **Tailwind not compiling:**\n   - Kill dev server: `Ctrl + C`\n   - Reinstall deps: `npm install`\n   - Restart: `npm run dev`\n   - Wait 30 seconds for compilation\n\n## 🌐 If Page Loads But Can't Login/Register\n\n1. Check backend is running:\n   ```bash\n   curl http://localhost:5000/api/health\n   ```\n\n2. Check CORS is enabled in backend\n\n3. Check .env in backend has:\n   ```\n   FRONTEND_URL=http://localhost:3000\n   ```\n\n## 📞 Still Stuck?\n\n**Provide this information:**\n\n1. What do you see? (blank page, error message, etc.)\n2. Browser console errors (F12 → Console)\n3. Network tab errors (F12 → Network)\n4. Terminal output when running `npm run dev`\n5. Is backend running? (Test http://localhost:5000/api/health)\n6. Is MongoDB running?\n\n## 💡 Pro Tips\n\n1. **Always use 2 terminals** - one for backend, one for frontend\n2. **Wait after npm install** - Vite can take 30 seconds to start\n3. **Check ports** - Don't have another app using 3000 or 5000\n4. **Fresh browser tab** - Try incognito/private mode\n5. **Check Task Manager** - Kill any node.exe processes before restarting\n\n---\n\n**Most Common Fix:** Delete node_modules and reinstall!\n\n```bash\nrmdir /s node_modules\ndel package-lock.json\nnpm install\nnpm run dev\n```\n\nWait 30 seconds after `npm run dev` starts, then refresh browser.\n
