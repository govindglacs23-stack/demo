# LMS - Complete Windows Setup Guide\n\n## 📋 Prerequisites\n\nBefore you start, ensure you have:\n\n### 1. Node.js & npm\n- Download: https://nodejs.org/\n- Choose the LTS version\n- After installation, restart Command Prompt\n- Verify: `node --version` and `npm --version`\n\n### 2. MongoDB\nChoose one option:\n\n**Option A: Local MongoDB (Windows)**\n- Download: https://www.mongodb.com/try/download/community\n- Run installer and follow setup wizard\n- Start MongoDB: Open Command Prompt and run `mongod`\n- It will run on `mongodb://localhost:27017`\n\n**Option B: MongoDB Atlas (Cloud - Recommended)**\n- Go to https://www.mongodb.com/cloud/atlas\n- Create free account and cluster\n- Get connection string like: `mongodb+srv://username:password@cluster.mongodb.net/lms`\n\n---\n\n## 🚀 Installation & Running\n\n### Method 1: Super Easy (Recommended)\n\n1. **Open Command Prompt** in the LMS folder\n2. **Double-click `RUN_LMS.bat`**\n3. Choose option `3` to install both\n4. Once complete, choose option `4` to run backend\n5. Open **another Command Prompt** in LMS folder\n6. Double-click `RUN_LMS.bat` again\n7. Choose option `5` to run frontend\n8. Visit http://localhost:3000\n\n### Method 2: Manual Installation & Running\n\n#### Installation\n\n**Backend Installation:**\n```bash\ncd C:\\Users\\gopal\\OneDrive\\Desktop\\LMS\\backend\ninstall.bat\n```\n\n**Frontend Installation:**\n```bash\ncd C:\\Users\\gopal\\OneDrive\\Desktop\\LMS\\frontend\ninstall.bat\n```\n\n#### Running\n\n**Terminal 1 - Backend:**\n```bash\ncd C:\\Users\\gopal\\OneDrive\\Desktop\\LMS\\backend\nrun.bat\n```\nOr manually:\n```bash\nnpm run dev\n```\n\n**Terminal 2 - Frontend:**\n```bash\ncd C:\\Users\\gopal\\OneDrive\\Desktop\\LMS\\frontend\nrun.bat\n```\nOr manually:\n```bash\nnpm run dev\n```\n\n---\n\n## ⚙️ Configuration\n\n### Backend Configuration\n\n1. Go to `backend` folder\n2. Open `.env` file with notepad\n3. Set your MongoDB URI:\n\n**For Local MongoDB:**\n```\nMONGODB_URI=mongodb://localhost:27017/lms\nJWT_SECRET=your_secret_key_here\nPORT=5000\nNODE_ENV=development\nFRONTEND_URL=http://localhost:3000\n```\n\n**For MongoDB Atlas:**\n```\nMONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/lms\nJWT_SECRET=your_secret_key_here\nPORT=5000\nNODE_ENV=development\nFRONTEND_URL=http://localhost:3000\n```\n\n4. Save the file\n\n### Frontend Configuration\n\nFrontend uses `http://localhost:5000` by default. If you changed the backend port, edit:\n- File: `frontend/src/services/api.js`\n- Line: `const API_BASE_URL = 'http://localhost:5000/api';`\n\n---\n\n## 🌐 Access the Application\n\nOnce both servers are running:\n\n- **Frontend:** http://localhost:3000\n- **Backend API:** http://localhost:5000/api\n- **Health Check:** http://localhost:5000/api/health\n\n---\n\n## 📝 First Time Setup\n\n1. Package.json has correct version\n- Delete `node_modules` folder and `package-lock.json`\n- Run `npm install` again\n\n### Port Already in Use\n\n**Error:** `Port 5000 or 3000 already in use`\n\n**Solution:**\n- Find and close the process using the port\n- Or change the port:\n  - Backend: Edit `backend/.env` → `PORT=5001`\n  - Frontend: Edit `frontend/vite.config.js` → `port: 3001`\n\n### npm command not found\n\n**Error:** `npm is not recognized`\n\n**Solution:**\n- Install Node.js from https://nodejs.org/\n- Restart Command Prompt after installation\n\n### CORS Error in Browser\n\n**Error:** `Access to XMLHttpRequest blocked by CORS`\n\n**Solution:**\n- Make sure backend is running\n- Check `FRONTEND_URL` in backend `.env`\n- It should be `http://localhost:3000`\n\n### Frontend can't reach backend\n\n**Error:** `Cannot connect to backend` or network error\n\n**Solution:**\n- Check backend is running on http://localhost:5000\n- Check browser console (F12) for error details\n- Verify API base URL in `frontend/src/services/api.js`\n\n---\n\n## 📂 Project Structure\n\n```\nLMS/\n├── backend/\n│   ├── config/            # Database config\n│   ├── controllers/       # Business logic\n│   ├── models/            # Database schemas\n│   ├── routes/            # API endpoints\n│   ├── middleware/        # Authentication\n│   ├── .env               # Configuration (created from .env.example)\n│   ├── .env.example       # Template\n│   ├── package.json       # Dependencies\n│   ├── server.js          # Entry point\n│   ├── install.bat        # Installation script\n│   └── run.bat            # Run script\n│\n├── frontend/\n│   ├── src/\n│   │   ├── pages/         # Page components\n│   │   ├── components/    # Reusable components\n│   │   ├── services/      # API service\n│   │   ├── App.jsx        # Main app\n│   │   └── main.jsx       # Entry point\n│   ├── package.json\n│   ├── vite.config.js\n│   ├── tailwind.config.js\n│   ├── install.bat\n│   └── run.bat\n│\n├── RUN_LMS.bat            # Main menu (use this!)\n├── START_HERE.md          # Quick start\n├── SETUP_WINDOWS.md       # This file\n├── README.md              # Full documentation\n└── QUICKSTART.md          # Quick reference\n```\n\n---\n\n## ✨ Features\n\n✅ **User Authentication** -
